# Test Data Management

Generate, seed, and clean up test data for your test suites.

## Parse Arguments

- `$0` = action: `generate`, `seed`, `cleanup` (required)
- `--schema <name>` = entity schema: `user`, `product`, `order`, `post`, `comment`, or custom
- `--count <N>` = number of records to generate (default: 10)
- `--output <file>` = output file path (default: `test-data/<schema>.json`)
- `--format <fmt>` = output format: `json` (default), `csv`, `sql`
- `--locale <loc>` = Faker.js locale (default: `en`)

## Steps

### 1. Action: `generate`

Generate test data and save to file.

**Install Faker.js if needed:**
```bash
node -e "require('@faker-js/faker')" 2>/dev/null || npm install -D @faker-js/faker
```

**Generate data using Node.js script:**

The skill should use the Write tool to create a generation script based on `--schema`, then execute it:

```bash
node -e "
const { faker } = require('@faker-js/faker');
const count = $COUNT;
const data = Array.from({ length: count }, (_, i) => ({
  // Schema-specific fields generated by the skill
}));
console.log(JSON.stringify(data, null, 2));
" > $output_file
```

**Built-in schemas:**

- `user`: id, name, email, phone, address, avatar, role, createdAt
- `product`: id, name, description, price, category, sku, stock, image
- `order`: id, userId, products[], total, status, shippingAddress, createdAt
- `post`: id, title, body, authorId, tags[], publishedAt, likes
- `comment`: id, postId, authorId, body, createdAt

For custom schemas: ask the user to describe the fields, then generate accordingly.

### 2. Action: `seed`

Insert generated data into the database.

**Detect ORM and run seed:**
```bash
# Prisma
npx prisma db seed

# Knex
npx knex seed:run

# Custom seed script
node seeds/seed.js
```

Or use the generated JSON file to seed via API:
```bash
node -e "
const data = require('./$output_file');
// POST each record to API
"
```

### 3. Action: `cleanup`

Remove test data.

```bash
# Delete generated files
rm -f test-data/*.generated.json

# Or reset database
npx prisma db push --force-reset
# or
npx knex migrate:rollback --all && npx knex migrate:latest
```

## Error Recovery

- If Faker.js not installed: auto-install with `npm install -D @faker-js/faker`
- If unknown schema: ask user to describe fields (name, type, constraints)
- If output directory doesn't exist: create it
- If database seed fails: check connection string and migration status
- For large datasets (>10000): suggest using streaming/batch approach
